## 1. Общая информация о БД

В проекте используется **одна реляционная база данных** — **PostgreSQL**. С ней работает веб-приложение на **Django**: описание таблиц задаётся **моделями** (Python-классы), физическая схема в PostgreSQL создаётся и обновляется **миграциями**.

**Что хранится в БД по смыслу:**

- **Служебные и платформенные данные Django** — пользователи и группы (`auth`), сессии браузера, записи админки, типы контента и др. (таблицы приложений из `INSTALLED_APPS`).
- **Предметная область QuizSlides** — сессии, участники, презентации, слайды, виджеты (опрос, облако слов), голоса и связанные сущности. Это модели приложения **`core`** (в БД таблицы с префиксом `core_`).

## 3. Подключение к БД

1.Установить адаптер - **pip install psycopg2-binary**

2. Настроить settings.py 
Пример: 
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': 'quizslides_bd',      # имя базы данных
        'USER': 'postgres',            # пользователь
        'PASSWORD': 'ваш_пароль',      # пароль
        'HOST': 'localhost',           # сервер (локальный)
        'PORT': '5432',                # порт PostgreSQL
    }
}

3. Создать Базу Данных - **CREATE DATABASE quizslides_bd;**

4. Применить миграции - **python manage.py migrate**

5. Проверка - **python manage.py runserver**


### 4 Таблицы приложения `core`

В БД создаются таблицы, соответствующие моделям Django (имена таблиц по умолчанию: `core_<имя_модели_в_нижнем_регистре>`).

| Модель (таблица) | Описание |
|------------------|----------|
| **Session** | Сессия викторины (код, статус, время, макс. участников, создатель) |
| **Member** | Участник сессии (имя, связь с сессией) |
| **Reaction** | Реакция на сессию (тип Cool / Bad, количество) |
| **Presentation** | Презентация (название, статус, связь с пользователем и сессией) |
| **Slide** | Слайд презентации (номер, картинка, статус) |
| **Widget** | Виджет на слайде |
| **WordCloud** | Облако слов (лимит слов, вопрос) |
| **AnswerOption** | Вариант ответа (номер, текст) |
| **Quiz** | Опрос / викторина (макс. время, возможность менять ответ) |
| **Vote** | Голосование (связь с участником, опросом, презентацией, виджетом) |

## 4. ER-диаграмма и описание связей

Ниже — **настоящая ER-схема** (таблицы, поля, ключи). Файл с рисунком должен лежать в репозитории по пути **`doc/images/er-schema.png`** — тогда изображение подтянется в предпросмотре Markdown.

![ER-схема базы данных QuizSlides](images/er-schema.png)

*На схеме сущность **User** показана в логическом виде (`id_user`, `login`, `password`). В текущем проекте на Django создатель сессий и презентаций — это стандартная модель **`auth.User`** (в БД таблица `auth_user`, набор полей см. в миграциях `auth`). Поле **`picture`** у слайда на схеме как BLOB; в коде Django используется **`ImageField`** — в таблице хранится путь к файлу, не двоичный объект.*

### 🔗 Описание связей (Cardinality)

| Связь | Тип | Пояснение |
| :--- | :--- | :--- |
| User → Session | 1 : N | Один пользователь может быть создателем нескольких сессий. В сессии `id_user` — внешний ключ. |
| Session → Member | 1 : N | В одной сессии может быть много участников (Member). |
| Session → Reaction | 1 : N | К одной сессии можно оставить много реакций (Cool / Bad). |
| Session → Presentation | 1 : N | В рамках одной сессии может быть несколько презентаций. |
| Presentation → Slide | 1 : N | Одна презентация содержит много слайдов. |
| Slide → Widget | 1 : N | На одном слайде может быть несколько виджетов (опросы, облака слов и т.д.). |
| Widget → Quiz | 1 : 1? / 1 : N | К одному виджету может быть привязан опрос (Quiz). На схеме — внешний ключ `id_widget` в Quiz. |
| Widget → WordCloud | 1 : 1? / 1 : N | Аналогично: виджет может содержать облако слов. |
| Widget → AnswerOption | 1 : N | У одного виджета (опроса) может быть несколько вариантов ответов. |
| User → Presentation | 1 : N | Пользователь может создавать много презентаций (`id_user` в Presentation). |
| Member → Vote | 1 : N | Участник может голосовать много раз (в разных опросах / виджетах). |
| Quiz → Vote | 1 : N | В одном опросе может быть много голосов. |
| Presentation → Vote | 1 : N | В рамках одной презентации может быть много голосов. |
| Widget → Vote | 1 : N | В одном виджете может быть много голосов. |

Типы связей согласованы с внешними ключами на ER-схеме: на стороне «много» стоит таблица, в которой хранится FK.

## 5. Взаимодействие приложения с БД

- Доступ к данным из кода — через **ORM Django** (`core/models.py`, пользователь — `auth.User`).
- Изменение схемы — команды **`makemigrations`** и **`migrate`**.
- Поле **`Slide.picture`**: в БД хранится путь к файлу, не само изображение.

## 6. Миграции `core`

Начальная схема домена: **`core/migrations/0001_initial.py`** (зависимость от `AUTH_USER_MODEL`).
